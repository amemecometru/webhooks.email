// 01 // INBOUND WEBHOOKS

RECEIVE WEBHOOKS
AT YOUR ENDPOINT.

Overview

Every endpoint gets a unique URL at https://you.webhooks.email/hook/{id}. Any HTTP method — GET, POST, PUT, PATCH, DELETE — is accepted, logged, and forwarded to your configured destinations.

Headers, body, method, and timestamp are captured for every request. The full audit log is searchable through the dashboard and API. Failed deliveries go to the dead letter queue with automatic retry.

curl / sender
→
you.webhooks.email /hook/{id}
→
forward
→
your server
POST / PUT / PATCH / DELETE capture + log transform + retry 200 OK

Quickstart

Create an endpoint, then send a webhook. Replace YOUR_API_KEY with your key from the dashboard.

SHELL 
# Create an endpoint
curl -X POST https://app.webhooks.email/api/endpoints \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-endpoint", "destinations": [{"url": "https://myserver.com/webhook"}]}'

# Response — save the webhook_url and api_key
{
  "id": "abc-123",
  "webhook_url": "https://you.webhooks.email/hook/abc-123",
  "api_key": "sk_...",
  "plan": "free"
}

# Send a webhook
curl -X POST https://you.webhooks.email/hook/abc-123 \
  -H "Content-Type: application/json" \
  -d '{"event": "invoice.paid", "amount": 4999, "currency": "usd"}'

# Response
{
  "status": "received",
  "delivery": "forwarded",
  "destinations": ["https://myserver.com/webhook"]
}

ENDPOINT

POST /hook/{id}

Accepts any HTTP method. Content-Type is auto-detected.

AUTH

No auth required to send webhooks. API key only needed for dashboard/management.

LIMITS

Free: 1,000/mo. Pro: 100,000/mo. Enterprise: 5M/mo.

RETENTION

Free: 7 days. Pro: 30 days. Enterprise: 90 days.

// 02 // EMAIL BRIDGE

SEND EMAIL →
RECEIVE WEBHOOK.

Overview

Every endpoint gets anything@you.webhooks.email. Send an email to that address and we convert it to a structured webhook POST to your destinations. Full MIME parsing: text body, HTML body, attachments (base64), headers, and SPF/DKIM status.

Use it for form submissions, customer support integrations, or any system that can send email but not HTTP webhooks.

gmail / outlook
→
anything@ you .webhooks.email
→
webhook
POST
→
your server
SMTP inbound MIME parse JSON payload delivered

Send any email to your endpoint's email address. The payload your server receives looks like this:

// 03 // AI TRANSFORM

REWRITE PAYLOADS
WITH AI.

Overview

Attach a transform prompt to any endpoint. Every incoming webhook is sent through an LLM (OpenRouter) before being forwarded. The prompt tells the model how to reshape the payload — filter fields, rename keys, enrich with derived data, or convert between formats.

Use it when senders emit bloated payloads and you only need 3 fields. Or when you need to normalize webhooks from different sources into a consistent schema.

webhook
sender
→
/hook/{id} with transform enabled
→
OpenRouter LLM rewrite
→
your server
transformed payload
raw payload prompt applied AI rewrite clean output

Quickstart

Set a transform prompt on your endpoint, then send a webhook. The response includes x-webhook-transformed: true.

SHELL 
# Set a transform prompt (Pro plan required)
curl -X PUT https://app.webhooks.email/api/endpoints/abc-123/transforms \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Extract only the id, status, and total fields. Rename total to amount. Convert status to lowercase.",
    "model": "openai/gpt-4o-mini"
  }'

# Response
{ "status": "created" }

# Send a webhook — it will be transformed before forwarding
curl -X POST https://you.webhooks.email/hook/abc-123 \
  -H "Content-Type: application/json" \
  -d '{
    "id": "in_123",
    "object": "invoice",
    "status": "PAID",
    "total": 4999,
    "currency": "usd",
    "customer_name": "Alice",
    "customer_email": "alice@example.com",
    "line_items": [...]
  }'

# What your destination receives:
{
  "id": "in_123",
  "status": "paid",
  "amount": 4999
}
# header: x-webhook-transformed: true
# header: x-webhook-model: openai/gpt-4o-mini

ENDPOINT

PUT /api/endpoints/{id}/transforms

Set prompt, model, and optional output_schema.

MODELS

Default: openai/gpt-4o-mini. Any OpenRouter model supported. Set via model field.

PRICING

Prompts billed at OpenRouter rates (~$0.15/1M input tokens). Counts toward your webhook volume.

ERROR HANDLING

If the LLM call fails, the original payload is forwarded and x-webhook-transform-error header is set.
