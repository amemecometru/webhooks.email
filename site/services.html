// 01
📥

WEBHOOK INBOX

Every account gets a dedicated webhook URL at you.webhooks.email/hook. Accept any HTTP method — GET, POST, PUT, PATCH, DELETE. Full header and body capture. Real-time log with search and payload inspection.

HTTP INGRESS
// 02
✉️

EMAIL TO WEBHOOK

Send email to anything@you.webhooks.email and we POST it to your configured destinations as a structured webhook. Full MIME parsing: text body, HTML body, attachments (base64-encoded), headers, and SPF/DKIM status. Each email becomes a webhook event your application can process.

EMAIL BRIDGE
// 03
📬

WEBHOOK TO EMAIL

Receive webhooks and we forward them as formatted email messages to any address. Headers become subject lines, payload body renders as formatted text, and delivery status is included. Set up email alerts from any webhook source in seconds.

REVERSE BRIDGE
// 04
🔄

WEBHOOK RELAY

Forward every incoming webhook to one or many destinations simultaneously. Automatic retry with exponential backoff (3 attempts by default, configurable). Delivery receipts. No data loss — we log the full request and response of every forwarding attempt.

FORWARDING
// 05
⚡

WEBHOOK TRANSFORM

Modify payloads before forwarding using our transformation engine. Filter fields, rename keys, inject headers, conditionally route based on content. Use pre-built templates (Stripe, GitHub, Clerk, Resend) or write custom jq-style transforms.

MIDDLEWARE
// 06
💀

DEAD LETTER QUEUE

When a destination is unreachable, the failed webhook is stored in the dead letter queue with full context: request, response, error, and timestamp. Replay individually or in bulk. Export to JSON. Configurable retry policies per destination.

RELIABILITY
// 07
📋

WEBHOOK FILTER

Route webhooks based on content. Define rules like "only forward if event.type == invoice.paid" or "block if payload contains sensitive keywords". Filters apply before forwarding, reducing downstream noise and cost.

ROUTING
// 08
⏰

WEBHOOK SCHEDULER

Schedule webhook delivery at specific times. Cron syntax. Use it for daily health checks, weekly report triggers, or integrating with time-based automation. Each scheduled delivery is logged and billed as a webhook.

CRON
// 09
🪞

WEBHOOK MIRROR

Duplicate incoming webhooks to multiple environments simultaneously. Send one copy to production, one to staging, one to the dead letter queue. Each destination receives the same payload with identical headers.

DUPLICATION
// 10
📤

BULK EXPORT

Export all captured webhooks as JSON, NDJSON, or CSV. Filter by date range, endpoint, delivery status, or content pattern. Useful for migration, audit, or analytics pipelines. API and dashboard access.

DATA
// 11
🔐

SIGNATURE VERIFICATION

Verify incoming webhook signatures automatically. Built-in support for Stripe, GitHub, Clerk, Svix, and custom HMAC schemes. Invalid signatures are flagged and optionally blocked before forwarding.

SECURITY
// 12
📊

WEBHOOK ANALYTICS

Dashboard with real-time metrics: throughput, delivery success rate, average latency, error breakdown by destination. Webhook volume charts. Export raw metrics via API for custom monitoring.

OBSERVABILITY

EMAIL AUTHENTICATOR

Verify email ownership via webhook callback. Send a verification email from verify@webhooks.email — clicking the link or replying triggers a webhook to your endpoint with the verification payload.
